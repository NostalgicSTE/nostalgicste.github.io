                    Continuation
                

                

                    

                        

                            
Atlassian

                            
Atlassian software suite is a fairly new concept for the most part, but I was already
                                sure about Github's "doppelganger", Bitbucket, through Intuition@NTU. In the context of
                                the internship, I ran instances of my own servers in virtual machines through which I
                                got to understand the capabilities of Atlassian suite. Learning curve was
                                notably steep, but with some motivation and guidance, I was able to get the hang of it.
                            

                            

                            
                        


                        

                            
Active Directory

                            
The process of creating service accounts for new company colleagues from various
                                departments into the company's server was tiring and repetitious for the most part, but
                                it ended quickly through the sincere effort of my friend, Azfar and I. Also got the
                                opportunity to work on Keycloak platform, an open-sourced access management suite, along
                                with Active directory, serving as a middleman between Jira and Active Directory, ran
                                across Nginx server to a Node.js application. It was all a new experience for me.

                            

                            
                        


                        

                            
Bitbucket

                            
Having used a similar product as Bitbucket, I instantaneous understood its workflow and
                                nuances of the platform. It was easy for me execute the migration of projects, which was
                                mostly automated by the software. The efforts come in during pre-migration period, where
                                we had to document user rights and permission for the existing projects on the old
                                server. It was a tedious yet an important portion of the migration process, where we
                                just could not afford to make mistakes, as it was actual projects which were at stake.
                            

                        


                        

                            
Network+

                            
I always loved to fantasize about networking systems, and the world of wireless
                                communications, since my secondary school days. Network+ by CompTIA is a great starting
                                point into this world, as it went through all from basics such as OSI (Open Systems
                                Interconnection) model and network topologies, to advanced concepts such as CIA Triad
                                and social engineering. I would say that this course was one of the key reasons I
                                applied for this internship.

                            

                            
                        

                        

                    

                    

                        

                            So many wonderful memories...

Linux

                                    
Linux is one of the best operating system I have worked with in the past few
                                        decades. It provides me with a level of freedom, that other OS(s) don't. At ST,
                                        I got to put knowledge to work, when we tested and ran Atlassian suite from Kali

Jira

                                    
Jira project migration was easier than Bitbucket, as there was little
                                        human
                                        intervention when importing the project over to the new environment, but
                                        involved
                                        more documentation. However, this migration is also the process with the most

                                        dissatisfactory user experience, it was messy.

                                


                                

                                    
Wrap-up

                                    
Following Jira migration, we had to address problems and verify migrated datas'
                                        authenticity, for Bitbucket and Confluence. This checking process was rather
                                        relaxed as there were no new issues nor hiccups.

                                

                            

                        

                    

                

            

        

    

    

        

            

                

                    Executionultra
                

                

                    


                        

                            
Server maintenance

                            
Working in a server room, and maintaining them has always been my dream, since I last saw
                                an engineer's experience video, at Google's data center. The experience I got during my
                                time at Amo Kio site's server room, fulfilled my long time desire. At there, we we
                                involved in various kinds of physical and technical works, from wire works in switches
                                and production servers, to installation and setup of various servers. 

                            
                        


                        

                            
Service desk application

                            
The service desk application was initiated by our senior intern, Bryan, and we got the
                                opportunity to contribute to it. We created a function that is to create issue request
                                to project admin through Jira platform's automation feature. This application uses runs
                                on Node.JS and is executed at the server running Active Directory.

                            

                            

                            

                            
                        


                        

                            
Active Directory User Retrieval

                            
This application, or rather process is a simple user information extractor that runs on
                                the Active Directory server. However the challenging part with this creation was that we
                                had to essentially create a Comma-Seperated-Value (CSV) file without having excel
                                installed in the production server. Initially we programmed our prototype PowerShell
                                (.ps1) script in a virtual machine, where there was excel pre-installed. Due to that, we
                                were able to use Excel-dependant commands. Moving over to the actual server was when we
                                had to purge the created script and redo a new one without using excel application. The
                                process of creating this script was honestly tiring and demotivating. But soon enough,
                                we managed to crack the code and were able to create a workable script.
